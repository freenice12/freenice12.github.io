github-2fa-이후-cli-작업하기기

2018-02-19
2fa, cli, github

사건의 발단

어느날 갑자기 Node.js 그룹에서 제명 되었습니다. 왜일까 했더니 Node.js에서 2FA(Two Factor Authentication)를 활성화 하지 않은 멤버는 자동으로 떨구기로 결정한것 이었습니다. ㅠㅠ

부랴부랴 2FA를 활성화하고 “나 다시 멤버로 등록시켜줘 횽들!” 이라고 외쳤고 결국 다시 멤버가 되었습니다. 만,!

잘 사용하고 있던 git이 cli에서 말썽을 부렸습니다. 기존 저장소에 푸쉬가 안되는 현상! 바로 깃 관리자가 바보가 된것입니다.

2FA를 활성화하자 기존 비밀번호를 냅다 때리는 것은 막히게 된것이죠. 그럼 어떻게 해야할까… 이 문서 저 문서 웹서핑을 하면서 삽을 마구 휘둘렀습니다.

이런 저런 문서를 검색하다가 (https://help.github.com/articles/providing-your-2fa-authentication-code/)와 (https://help.github.com/articles/working-with-ssh-key-passphrases/)를 보게 되었고 SSH를 재 활성화(?) 하는 쪽으로 가닥을 잡았습니다. 도데체 알 수 없는 passphrase와 함께요. ㅠㅠ

위 명령어를 입력하면 새 passphrase를 입력하라고 나옵니다(문서에는요). 하지만 저는 기존 passphrase를 입력하라는 창이 떳습니다. 아직도 모르겠습니다. 저는 최초 ssh key를 등록할 때 passphrase를 설정한 적이 없구던요 ㅠㅠ이런 문서도 ㅠㅠ

그럼 방법은 다른 하나! 새로 만든다!

먼저 SSH key를 하나 새로 만들었습니다.

이때 2FA를 활성화 했다면 passphrase를 입력해줘야 합니다. 2FA용 비밀번호(?)로 말이죠… 이 키는 어디서 얻나! 바로 깃허브 세팅에 있습니다. 셋팅이 조금 바껴있을 꺼에요 아마도?

위 메뉴에 들어가면 새 토큰을 발급받을 수 있습니다. 발급받은 당시만 보여주니까 잘 간직해주세요! 물론 콘트롤(커맨드) + C!

이제 복사한 토큰을 passphrase에 붙여넣고 SSH key를 마져 생성합니다. 생성한 rsa는 깃허브 세팅에 다시 추가해주시고요.

그럼 끝!

참쉽죠!?
