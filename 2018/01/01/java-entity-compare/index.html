아… 원래 없던 감도 사라졌습니다.

원래 하려던 테스트는

  • 메타 데이터 받음
  • 받은 메타 데이터로 새로운 객체 생성
  • 새로운 객체가 메타 데이터를 사용했는지 판별

위 3단계 입니다. 하지만 몇 번의 테스트를 실패하고서 떠오른 내용이 있었으니…
객체 비교 테스트를 할 때는 hashCode랑 equals 메소드를 오버라이드 해줘야 한다!

이런 간단한 내용조차 삽을 들어야 한다니… 많이 슬픕니다.

문제가 된 내용은 이렇습니다.

두 객체는 엄연히 다른 객체입니다. 하지만 우리가 JPA나 도메인에서 비교하려는 내용은 객체가 어디를 참조하고 있는지가 아닌! 객체의 내용이 같은지 입니다.

이 점을 상기한 후 아래처럼 hashCode, equals 메소드를 오버라이드 했(IDEA가 ㅋㅋ)습니다.

id는 필수가 아닌 상태이고(이제 만들기 시작했으니…) 정말 들어갈 수 있는 내용은 현재 email뿐이라고 가정한 코드입니다. 물론 제가 짜라고 하면 못짭니다…TT

요즘 기본기가 탄탄치 않아 민망한 상황이 자주 발생하는데 더 집중 해야겠습니다(정신적으로 힘든 상태 핑계대기!). 어뎁터, 팩토리, 전략 패턴등에 대해 글을 써보고싶은 느낌이네요(한차례 휩쓸고간 민망함 때문에 정리하고 싶은 욕구!).

암튼 그렇습니다.

끘!
