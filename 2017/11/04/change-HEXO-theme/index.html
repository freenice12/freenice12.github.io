hExO-테마-변경하기

2017-11-04
hexo, theme

오늘은 헥소 테마를 변경하도록 하겠습니다. 사실 기본 테마가 굉장히 이뻐보였는데 다른 테마를 좀 봤더니(그리고 다른 불로구!) 역시 디폴트로는 만족할 수 없었습니다.

0단계: 테마 고르기

테마 여행을 좀 해보니 두 개의 후보군이 있었고 최종적으로 좀 더 많이 보였던? heuman을 사용하기로 마음 먹었습니다.

1단계: 테마 적용하기

테마 설치 & 설정

테마를 정했다면 블로깅 폴더의 /themes 폴더에 들어가 원하는 테마를 클론 받습니다. 그리고 해당 테마의 설명대로 설정들을 수정해줍니다(heuman 기준).

위 설정 파일은 수정하는 족족 서버에 반영됩니다. 물론 깃에 푸쉬해야 하는 경우에는 안되겠죠?!

메인 설정의 테마 변경

이제 마지막으로 루트 디렉토리에 있는 설정파일을 변경합니다.

설정파일에서 theme: landscape(아마도 기본이라면) 로 된 부분을 theme: hueman으로 변경해줍니다.

테마 누리기

아무 것도 안뜰것 같습니다. 왜그럴까? 수정파일을 모두 변경해봐도 도저히 답을 찾을 수 없었습니다. 이렇게 2일이 지나고… 오늘 마음을 가다듬고 설정을 한땀한땀 관리하다 보니…

아………………………. ㅠㅠ
그렇습니다… 오타가 이렇게 무섭습니다. 여러분!!!! 내 시간!!!!
heuman으로 정정한 후에 역시나 정상적으로 돌아가기 시작합니다.

마무으리

후로그래머가 아니라도 오타를 찾아내고 정정하는 일은 매우 중요합니다. 남탓하지 않는 내가 되기위해서 오늘도 노력합니다!

당장은 heuman이 제공하는 기본이긴 하지만 이것도 슬슬 바꿔나가려 합니다. 중요한건 글들이니까요…(개인적으로 ㅎㅎ)
그리고 기본 설정중에 thumbnail이 있는데 기본이 보이는거라 false로 변경했습니다.

문제점 발견!! - 추가

기본 모드에선 몰랐던 한글 파일의 페이지를 클릭하면 404가 뜨는 문제가 발견되었… 그렇다면 파일을 영어로 바꾸고 제목을 한글로 다는수 밖에… 어떤 존잘님이 해결해주시기 전까진 해결방법을 계속 찾아 봐야겠습니다(쫄병은 괴로워T.T).

추가 2

일단 근본적으로 hexo의 문젠 아닌것 같고 git의 문제로 파악.

위 옵션 실패!

추가 3

결과적으로 아래 옵션도 실패했으며, 한글 파일명을 사용하지 않는게 이롭다는 결론을 내려준 블로그를 보고 슬슬 마음을 접었습니다.

추가 4

위에서 말하듯 한글 파일명을 사용하지 않는것이 정신 건강에 이로울것 같습니다.
일단 이 블로그에서 어느정도의 해결법을 찾았으나 해보진 않는걸로 마무리 했습니다(이제 진짜 끘!).

추가 - disqus 설정

컨피그 파일에 comment 영역을 보면 disqus 설정부가 있습니다. 자기 disqus의 shortname을 쓰라고 나오는데요. disqus 페이지에서 가입을 한 후

  • 아바타를 클릭 (profile 페이지 이동)
  • 이름 옆 Edit Profile 클릭
  • 왼쪽 메뉴 중 Account 클릭
  • Username을 설정 파일에 입력
